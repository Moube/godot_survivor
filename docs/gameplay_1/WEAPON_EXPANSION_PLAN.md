# 武器扩展规划

本文档记录幸存者类武器扩展方向，以及逐个实现与验收节奏。

## 美术通用规则

- 新武器优先使用清晰的大轮廓和少量高识别色，不追求复杂花纹和华丽细节。
- 武器仍按当前项目规则表现为浮游法器、自动战斗物件或局内战斗能力，不要求玩家手持。
- 武器不一定需要可见的漂浮发射器。只有需要表达独立朝向、环绕位置或持续存在物件的武器，才制作漂浮发射器资源。
- `ProjectileEmitter` 表示“发射投射物的代码行为”，不等同于美术上一定有一个漂浮在玩家周围的发射器。
- 每个武器都必须考虑是否有可直接作为 icon 的资源。
- 若运行时资源本身适合 icon，应复用运行时资源；若运行时资源是横向动画 strip 或过小投射物，则单独制作 `asset/art/item/weapon_xxx.png`。
- icon 必须同时服务升级选项和 HUD 武器栏，在 44x44 左右显示时仍能识别。

## 实现节奏

按以下顺序逐个实现。每个武器完成代码、配置、美术和基础运行验证后，再进入下一个武器。

1. 星芒飞刃
2. 圣水符瓶
3. 抗拒火盾
4. 雷霆法印
5. 灵魂锁链

## 武器方案

### 1. 星芒飞刃

- 状态：实现中
- 类型：投射物武器
- 行为：无可见漂浮发射器，直接从玩家当前位置朝最后一次移动方向发射银蓝飞刃；飞刃可穿透少量敌人。
- 升级方向：增加伤害、降低冷却，后续等级增加发射数量。
- 美术：简化银蓝月牙飞刃，保留清晰外轮廓和少量蓝色内光，不制作发射器动画，不使用复杂星纹。
- icon：使用单独的简化飞刃 icon，路径规划为 `res://asset/art/item/weapon_star_blade.png`。

### 2. 圣水符瓶

- 状态：实现中
- 类型：地面持续范围
- 行为：无可见漂浮发射器，按独立冷却在敌人附近生成一个从天旋转下落的圣水瓶；瓶子落地后生成短时伤害水洼。没有敌人时，在玩家最后一次移动方向前方落下。
- 升级方向：增加范围、持续时间、伤害或投掷数量。
- 美术：小玻璃符瓶 icon + 下落符瓶 + 简单青绿色水洼圆区；水洼上使用烘焙后的整片低矮蓝火动画 `holy_water_flame_field_strip_6f.png`，运行时只播放一个火焰区域 Sprite；单个 Polygon 火苗场景保留给少量独立火苗复用，瓶子落地后干净消失并立即生成圣水区域。
- icon：使用单独的简化符瓶 icon，路径为 `res://asset/art/item/weapon_holy_water.png`。

### 3. 抗拒火盾

- 状态：实现中
- 类型：环绕物
- 行为：若干火球围绕玩家以较大半径旋转，接触敌人造成伤害，避免和发射器类武器的近身轨道重叠。
- 升级方向：增加火球数量、旋转速度、伤害或命中频率。
- 美术：简化橙红火球，带少量黄色内核和火盾感外圈，不做复杂火焰细节。
- icon：火球资源可直接作为 icon，路径为 `res://asset/art/item/weapon_repulsion_fire_shield.png`。

### 4. 雷霆法印

- 状态：待实现
- 类型：定点打击
- 行为：锁定敌人脚下，短延迟后落雷造成范围伤害。
- 升级方向：增加落雷数量、伤害、范围或降低延迟。
- 美术：简化紫白圆形预警法印 + 竖直雷柱，预警圈需要比装饰符文更清楚。
- icon：使用简化雷电符号或小法印 icon，不直接使用雷柱特效。

### 5. 灵魂锁链

- 状态：待实现
- 类型：链式伤害
- 行为：连接最近敌人，伤害跳转到附近目标。
- 升级方向：增加跳转次数、伤害、链条距离或降低冷却。
- 美术：简化紫色魂链，节点用小魂灯或短链段表达，避免复杂烟雾。
- icon：使用魂灯或短链段作为 icon，优先保证小尺寸识别。

## 当前验收标准

单个武器完成时至少满足：

- 配置表能加载，无缺失资源路径。
- 升级池能提供新武器和已有武器升级。
- HUD 武器栏和升级选项能显示正确 icon。
- 武器能在关卡内自动触发并造成伤害。
- 运行时无 Godot 报错。
