# UI 资产规格说明样本

本文档用于记录单个游戏 UI 美术贴图在生成前应明确的规格。后续为每个 UI 资产生成贴图时，建议先按此格式补全信息，再交给 Codex 生成或迭代。

## 样本：主菜单普通按钮背景

```text
资产：主菜单普通按钮背景
用途：Godot TextureButton，支持 hover/pressed/disabled 状态
尺寸：512x128
风格：半写实奇幻 UI，深铁灰金属边框，低饱和蓝绿色内底
材质：细微金属磨损，边缘高光克制
透明背景：需要
禁止：不要文字、不要图标、不要强烈发光、不要卡通糖果色、不要水印
可九宫格拉伸：边框厚度约 24px，中间区域尽量平整
```

## 使用建议

- 先用少量代表性资产锁定整体风格，再扩展整套 UI。
- 同一类资产应保持统一的边框厚度、材质、色彩和光照方向。
- 需要在 Godot 中缩放或拉伸的贴图，应优先考虑 `NinePatchRect` 或九宫格切分规则。
- 如果资产包含不同交互状态，应分别说明普通、悬停、按下、禁用状态的视觉差异。
- 每次迭代只修改一个主要问题，例如边框厚度、亮度、材质强度或透明边缘质量。
