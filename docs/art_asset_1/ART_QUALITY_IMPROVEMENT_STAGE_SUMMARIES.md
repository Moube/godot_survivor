## 阶段 2：灰模、配色与场景基底验证完成记录

记录日期：2026-04-25

### 阶段目标

阶段 2 的目标不是制作最终正式插画，而是用低细节、统一色块和简单形状验证场景基底是否足够干净、统一，并确认玩家、敌人、子弹在真实游戏截图中的识别空间是否被释放出来。

本阶段重点解决三个问题：

1. 地面过于抢眼，干扰战斗主体。
2. 障碍物风格与场景不统一，且阻挡关系不够清楚。
3. 边框存在感过强，容易把战斗区变成装饰框。

### 本阶段完成内容

#### 1. 完成低细节地面 tile

本阶段替换了原有地面资源，最终使用：

- `asset/art/level/floor_tile.png`

地面采用柔和草绿色、低纹理密度和弱块面变化，重点是作为战斗背景，而不是成为视觉中心。

执行过程中曾发现第一版 tile 拼接存在轻微黑色缝隙，因此迭代了无缝版本，并通过游戏内截图确认：

- tile 与 tile 之间没有明显接缝。
- 地面纹理足够安静，不再压过玩家、敌人和子弹。
- 画面整体明亮，但没有依赖高对比纹理制造廉价复杂感。

#### 2. 将障碍物改为细长遗迹石柱

本阶段放弃了早期石砖块方向，改为更符合场景基底规则的细长遗迹石柱，最终使用：

- `asset/art/level/obstacle_pillar.png`
- `scene/level/ObstaclePillar.tscn`

最终确认的障碍物方向：

- 画布设计尺寸为 `20x80`。
- 场景内等比放大 `2x` 使用。
- 视觉上是细长遗迹石柱，而不是粗重石块。
- 保留轻微 3/4 顶面和少量前侧 / 右侧厚度。
- 表面细节控制在低密度，只保留基础分块、裂纹和磨损。
- 左上来光，阴影朝右下。

同时调整了碰撞与排序基准：

- 碰撞体只覆盖石柱底部接地部分，不覆盖整张贴图高度。
- 石柱放入 `World` 节点并启用 y-sort，以底部接地区域作为前后遮挡判断依据。
- 玩家或敌人在石柱前方时，应显示在石柱上方；在石柱后方时，应被石柱遮挡。

这个结构解决了纯 2D 场景中高物体遮挡的问题，不需要改成 3D 场景。

#### 3. 新增可复用 2D 投影阴影组件

为了避免每个资源都用手工 `Polygon2D` 画阴影，本阶段新增了可复用的程序化阴影组件：

- `scene/common/DropShadow2D.tscn`
- `script/common/DropShadow2D.cs`
- `asset/art/effects/drop_shadow_2d.gdshader`

该组件当前已经用于遗迹石柱，并被设计为后续玩家、敌人等 Sprite2D 资源也可以复用。

最终阴影结论：

- 阴影由接地点阴影和投影阴影组成。
- 阴影颜色使用偏草地环境的暗绿灰，而不是纯黑。
- 阴影方向统一为右下，符合左上来光。
- 阴影应与物体底部接触，不应漂浮或与接地点脱节。
- 阴影强度中等，能建立体积感，但不应压暗整体画面。

#### 4. 将边框改为抽象弱提示线

本阶段最终确认边框不再使用明确石质材质，而是改为抽象、非物质性的边界提示线，最终使用：

- `asset/art/level/wall_border_line.png`

边框处理结论：

- 边框只负责提示关卡边界，不承担装饰作用。
- 线条比最初版本更清楚，但仍明显弱于玩家、敌人、子弹和障碍。
- 横向与纵向边界共用同一张贴图，纵向边界通过 UV 旋转 / 映射实现，不再维护独立纵向资源。
- 边框不使用强石材属性，避免破坏轻微斜俯视 / 3/4 场景感。

#### 5. 清理并稳定最终资源命名

阶段完成后，清理了未使用的草稿资源和旧边界资源，并将最终资源改为不带阶段尾缀的生产命名。

当前阶段 2 相关的最终场景基底资源为：

- `asset/art/level/floor_tile.png`
- `asset/art/level/obstacle_pillar.png`
- `asset/art/level/wall_border_line.png`

清理：

- `scene/level/WallBlock.tscn`
- `asset/art/level/wall_tile_01.png`

清理原因是旧基础墙体场景已不再作为正式墙体实现使用。`Level01.tscn` 已改为直接定义墙体节点，并继续使用 `wall_border_line.png` 作为墙体显示资源。

### 验收结果

本阶段通过了截图验收，最终游戏内效果满足阶段 2 的核心目标：

1. 地面低细节、无明显拼接缝隙，背景不再抢夺主体注意力。
2. 遗迹石柱比地面更明确，但没有比玩家、敌人和子弹更抢眼。
3. 石柱排序关系符合纯 2D 俯视游戏需求，前后遮挡能根据接地点变化。
4. 石柱碰撞体只覆盖底部区域，符合高物体在 2D 中的阻挡直觉。
5. 阴影方向、颜色和位置已经与左上来光规则基本一致。
6. 边框能提示场地范围，但不再形成强装饰框。
7. 场景整体更干净，明显为玩家、敌人、子弹和后续战斗特效留出了识别空间。

### 本阶段形成的执行结论

1. 单张资源不能脱离游戏截图判断质量，尤其是地面、边框和阴影。
2. 带高度的 2D 障碍物必须拆分“视觉贴图高度”和“底部接地碰撞 / 排序基准”。
3. 3/4 感不应只靠障碍贴图实现，地面、阴影、边界和排序结构必须共同服务同一视角。
4. 边框如果具备强材质属性，会破坏场景倾斜感；阶段 2 后续应继续使用抽象边界线方向。
5. 阴影组件应优先复用，而不是每个资源单独画 polygon 阴影。
6. 资源定稿后应去掉阶段尾缀，使用稳定生产命名，便于后续阶段继续引用。

### 后续阶段注意事项

进入阶段 3 前，应维持阶段 2 的场景基底稳定，不再同时重做地面、边框、障碍和角色。

阶段 3 如果开始重做玩家、敌人、子弹、命中特效等核心资产，应遵守以下约束：

1. 不要让新角色资源重新压低地面和边框的可读性收益。
2. 玩家和敌人的阴影可以优先复用 `DropShadow2D`，但需要按体型单独调参数。
3. 如果角色资源加入更强描边或更高饱和色，应通过截图确认它们仍然明显高于背景优先级。
4. 不要为了让场景更“丰富”而在地面增加大量装饰，阶段 2 的干净基底应继续保留。
