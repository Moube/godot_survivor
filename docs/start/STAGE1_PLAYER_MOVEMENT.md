# 阶段1：玩家基础移动完成记录

## 阶段目标

本阶段的目标是建立最小可玩的基础流程：

1. 启动项目后先进入开始菜单
2. 点击开始后进入实际游玩关卡
3. 在关卡中加载玩家
4. 验证玩家移动、镜头跟随与基础碰撞

## 本阶段完成内容

### 1. 主流程结构调整

本阶段没有继续采用“`Main.tscn` 直接承载玩家”的做法，而是整理为两层结构：

- `scene/main/Main.tscn`
  作为开始菜单入口场景
- `scene/level/Level01.tscn`
  作为实际游玩的第一关场景

点击开始按钮后，由 `Main.cs` 切换到 `Level01.tscn`。

### 2. 玩家场景完成基础搭建

`scene/player/Player.tscn` 已完成以下基础结构：

- 根节点：`CharacterBody2D`
- 子节点：`CollisionShape2D`
- 子节点：`Camera2D`
- 临时可视节点：`Polygon2D`

其中临时可视节点用于在缺少正式美术资源时快速验证移动效果。

### 3. 输入映射完成

已配置以下输入：

- `move_up`：`W`
- `move_down`：`S`
- `move_left`：`A`
- `move_right`：`D`
- `fire`：鼠标左键

其中 `fire` 在本阶段仅作为后续阶段预留输入，不参与当前玩法逻辑。

### 4. 玩家移动逻辑完成

`scene/player/Player.cs` 已实现基础移动逻辑：

- 在 `_PhysicsProcess` 中读取输入方向
- 使用 `Input.GetVector()` 组合方向输入
- 通过 `Velocity` 赋值控制移动
- 使用 `MoveAndSlide()` 处理角色位移与碰撞

当前实现支持 2D 俯视角常见的四向/八方向移动表现。

### 5. 关卡内实例化玩家

`scene/level/Level01.cs` 负责在关卡启动时加载玩家：

- 通过导出的 `PlayerScene` 引用 `Player.tscn`
- 从 `PlayerSpawn` 出生点实例化玩家
- 将实例加入当前关卡场景树

这种做法使菜单、关卡、玩家职责分离，便于后续扩展多关卡、重生点和流程控制。

### 6. 临时可视化与镜头验证

为方便在没有正式贴图的情况下验证玩家移动，本阶段加入了两类临时可视化元素：

- 玩家自身使用简单占位图形显示
- 关卡使用网格背景辅助观察位移和镜头跟随

这样即使相机跟随玩家，也能肉眼确认玩家确实在世界中移动。

### 7. 碰撞验收环境完成

为了完成本阶段的碰撞验收，`Level01.tscn` 中加入了：

- 四周边界墙
- 关卡内部的多个测试障碍物

这些节点使用 `StaticBody2D + CollisionShape2D + Polygon2D` 的方式构建，既能参与碰撞，也能直接看到障碍位置。

## 最终验收结果

本阶段已完成并通过以下验收项：

1. 运行项目后先进入开始菜单
2. 点击开始按钮后成功进入关卡
3. 玩家可以稳定移动
4. 镜头跟随玩家正常
5. 玩家不会穿过边界墙和测试障碍物
6. 玩家贴边移动时表现正常
