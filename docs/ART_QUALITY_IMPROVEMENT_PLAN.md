# 游戏美术质感提升计划

## 文档目的

本文档用于规划当前 Top-Down Shooter Demo 的美术质感提升路径。
目标不是一次性生成整套正式美术，而是按阶段逐步提升画面统一性、战斗可读性与整体完成度。

执行原则：

1. 先统一视觉规则，再批量生成资源。
2. 先解决“廉价感来源”，再补细节。
3. 优先提升战斗可读性，不让背景和装饰抢主体。
4. 每一阶段都应可单独验收，再进入下一阶段。
5. 文档中的待定项必须先确认，再由 Codex 继续落地。

---

## 当前问题总结

结合当前游戏截图，现阶段画面问题主要集中在以下几个方面：

1. 风格来源不统一，背景、障碍、角色、UI 像来自不同素材体系。
2. 场景背景过于抢眼，削弱了玩家、敌人、浮游武器和魔法能量弹的识别效率。
3. 角色、障碍、UI 的尺度语言不一致，导致整体像拼贴。
4. 明暗、描边、材质处理缺少统一规则，造成低质感和廉价感。
5. UI 与场景世界观脱节，缺少统一的视觉系统。

## 当前已确认的视觉方向

以下内容已经在本轮规划中确认，可作为后续所有美术生成与替换工作的统一依据。

### 总体风格

- 整体视觉方向：明亮可爱卡通风
- 视角表现：轻微斜俯视，带一点 3/4 感
- 角色气质：可爱中带一点小帅气、小冒险感
- 描边强度：明显粗描边，强化卡通感
- 光照表现：整体明亮，但保留一定体积感和层次
- 纹理密度：低纹理、大色块、整体干净
- 整体饱和度：中等饱和，更柔和耐看

### 玩法可读性

- 场景背景存在感：尽量低调，优先服务战斗可读性
- 角色体量占屏比例：中等比例，兼顾场景空间感
- 玩家与敌人的配色区分：强区分，确保一眼分辨
- 魔法能量弹与战斗物件存在感：明显，保证飞行物和战斗节奏清楚
- 场景边框/围栏表现：弱化处理，仅作为边界提示

### 题材与材质方向

- 地面材质方向：柔和的泥土/草地混合地面
- 障碍物材质方向：细长遗迹石柱类奇幻障碍
- 特效风格强度：明亮夸张，强化命中与死亡反馈

### UI 方向

- UI 风格：简洁、低装饰、以清晰易读为主
- HUD 布局气质：极简，以数字和血条清晰为第一优先级

### 主角主色

- 玩家主色倾向：蓝色系
- 玩家辅助色倾向：白色 / 浅灰，走干净清爽路线
- 基础敌人主色倾向：红橙系，直接强调危险感
- 基础敌人辅助色倾向：深灰 / 黑灰，增强危险轮廓

## 总体阶段路线

本计划使用以下 9 个阶段作为主线。阶段编号表示美术观感提升的推进顺序，不等同于资源分类。

1. 阶段 1：先锁视觉规则，不生成新图
2. 阶段 2：先做灰模和配色，不急着做正式插画
3. 阶段 3：优先重做 6 个核心资产，不要全量铺开
4. 阶段 4：先统一“形”，再统一“纹理”
5. 阶段 5：场景层面做减法
6. 阶段 6：重新定义角色可读性
7. 阶段 7：补统一的反馈特效
8. 阶段 8：最后再处理 UI，而不是最早
9. 阶段 9：建立 Codex 执行方式，不要直接“帮我做一套美术”

当前已确认内容的归属：

1. 风格、尺寸、配色、描边、阴影等规则归入阶段 1。
2. 地面、障碍、边框的低细节基底规则归入阶段 2。
3. 6 个核心资产的正式重做顺序归入阶段 3。
4. 场景减法、背景降噪和边框弱化归入阶段 5。
5. 命中、死亡等反馈规则归入阶段 7。
6. HUD、血条、分数、Game Over 规则归入阶段 8。

## 阶段 1：先锁视觉规则，不生成新图

### 阶段目标

在开始新一轮资源生成前，先建立一套足够具体、可执行、可验收的视觉规则，避免后续继续出现“素材单张看似可用，但放在游戏里整体很劣质”的问题。

这一阶段不追求大量出图，重点是把“该生成什么样的图”定义清楚。

### 本阶段要完成的工作

1. 基于本文件已确认内容，补全一版更严格的视觉风格规范。
2. 把角色、敌人、地面、障碍、浮游武器、魔法能量弹、特效、HUD 的视觉职责拆清楚。
3. 明确后续资源生成的优先级，不允许同时大范围铺开。
4. 明确哪些资产必须先做“低细节统一版”，哪些内容可以延后 polish。
5. 形成可直接交给 Codex 使用的执行约束。

### 阶段产出物

本阶段结束时，应具备以下产出：

1. 一份更新后的视觉风格基准文档
2. 一份核心资产优先级列表
3. 一份资源生成约束清单
4. 一份后续阶段执行顺序说明

### 视觉执行基准

后续生成资源时，必须遵守以下基准：

1. 主体优先
   玩家、敌人、浮游武器、魔法能量弹、命中特效必须比背景更容易被第一眼识别。

2. 背景克制
   地面与边框不得使用过高对比、过强纹理、过亮高光。

3. 大色块优先
   角色、敌人、浮游武器、魔法能量弹、障碍优先用明确外轮廓和大色块表达，不以复杂纹理堆细节。

4. 单一世界观
   障碍、地面、特效、HUD 不能再分别走不同风格体系，必须统一到“明亮可爱奇幻卡通”框架下。

5. 统一光照逻辑
   后续资源要使用一致的体积处理与阴影逻辑，避免有的写实、有的平涂。

6. UI 从属于玩法
   UI 不追求装饰感，不走木牌、卷轴、宝石堆砌路线，优先做清晰、克制、易读。

### 本阶段建议先细化的视觉规范项

以下项目应在后续小步确认中逐项补全，并写回风格文档：

1. 玩家、敌人、浮游武器、魔法能量弹、障碍的推荐像素尺寸
2. 玩家与敌人的主辅配色关系
3. 地面与障碍的明度范围
4. 描边颜色与粗细规则
5. 阴影的方向、透明度与大小规则
6. 特效的颜色范围与持续时长感受目标
7. HUD 字体、字号、留白与布局规则

这些内容中，未确认的部分不能直接由 Codex 自由发挥生成正式资源。

### 已确认的尺寸基准

当前已确认的核心角色设计尺寸如下：

1. 玩家标准设计尺寸：`48x48` 左右
2. 基础敌人标准设计尺寸：略小于玩家，建议落在 `40x40` 到 `44x44` 左右
3. 魔法能量弹标准设计尺寸：`12x12` 左右
4. 常规单根障碍物画布设计尺寸：`20x80`
5. 障碍物底部接地 / 碰撞参考尺寸：`14x18`

执行约束：

1. 玩家必须在实际游玩视角下保持清晰轮廓和稳定识别。
2. 基础敌人可以略小于玩家，但不能小到削弱威胁识别。
3. 魔法能量弹要保证飞行方向易读，不应小到在战斗中被背景吞掉。
4. 遗迹石柱应保持细长比例，不能重新长成粗石块或方砖墙。
5. 障碍物底部接地面积应只覆盖石柱真正落地的部分，不把整张贴图外轮廓都算进碰撞。
6. 后续若出现精英敌人或大型敌人，应单独定义尺寸档位，不沿用基础敌人标准。

### 已确认的主辅配色关系

当前已确认的核心对立配色如下：

1. 玩家主色使用蓝色系
2. 玩家辅助色使用白色 / 浅灰
3. 基础敌人主色使用红橙系
4. 基础敌人辅助色使用深灰 / 黑灰

执行约束：

1. 玩家与敌人必须在第一眼视觉识别上形成明显冷热对比。
2. 玩家辅助色应服务轮廓清晰和干净气质，不引入过多装饰色。
3. 基础敌人的危险感优先于“配色丰富”，避免出现多色拼贴感。
4. 敌人的深色辅助色应主要服务轮廓和体块，不应把敌人整体压得过暗。

### 已确认的场景与表现规则

当前已确认的场景、描边、阴影与 HUD 规则如下：

1. 地面明度倾向：中等明度，更稳妥
2. 地面饱和度倾向：中低饱和，保留一点生气
3. 障碍物明度倾向：偏暗一些，强调厚重和阻挡感
4. 障碍物饱和度倾向：低饱和，明显退后
5. 描边颜色倾向：接近黑灰色，轮廓更强
6. 描边粗细档位：偏粗，卡通感明显
7. 阴影方向：左上来光，阴影偏右下
8. 阴影强度：中等，能看出层次但不压暗整体
9. HUD 字体气质：圆润可爱，轻松一点
10. HUD 数字信息位置倾向：右上角，传统稳妥
11. 血条表现方式倾向：底部居中横向血条，最直接

执行约束：

1. 地面必须比角色和特效更安静，但不能灰到失去生气。
2. 障碍物要有厚重感，但不得因为过暗而让轮廓糊成一片。
3. 描边应统一为稳定的黑灰轮廓，不对不同资产随意切换。
4. 阴影的主要职责是提供体积和落地感，不应把整体气氛压成偏沉重方向。
5. HUD 视觉优先级低于战斗主体，但信息读取优先级必须足够高。

### 已确认的战斗特效与 HUD 细化规则

当前已确认的战斗反馈与 HUD 细化规则如下：

1. 命中特效主色倾向：黄白色，经典直接
2. 墙体命中特效：与敌人命中特效区分开，敌人命中更亮，墙体命中更克制
3. 敌人死亡特效表现类型：碎片散开感
4. 敌人死亡特效持续感受：很短促，干脆利落
5. 枪口火花：暂不纳入第一轮，先把命中和死亡反馈做好
6. 血条显示方式：不显示数字，只保留图形血条
7. 分数字号存在感：中等，清楚但不抢画面
8. Game Over 面板视觉强度：偏弱，尽量保持整体克制

执行约束：

1. 战斗特效应优先强化“击中感”和“死亡确认感”，不追求复杂演出。
2. 敌人命中特效与墙体命中特效必须一眼可分，避免反馈语义混乱。
3. 死亡特效需要短促明确，不能拖慢当前战斗节奏。
4. 第一轮 HUD 仍以信息清晰为第一目标，不引入多余数字和装饰。
5. Game Over 面板即使偏弱，也必须保证状态转换足够明确，不能弱到看不出结束。

### 后续资源优先级

完成本阶段后，后续资产生成必须按以下顺序推进：

1. 地面与障碍
2. 玩家、浮游魔法短杖、敌人、魔法能量弹
3. 命中特效与死亡特效
4. HUD 基础面板与血条
5. 额外按钮、图标、装饰资源

这样做的原因：

1. 先把场景压安静，才能正确评估主体表现。
2. 先让战斗主体变清楚，才能判断玩法观感是否提升。
3. UI 与装饰资源放后做，可以避免前期风格再次漂移。

### 本阶段验收标准

达到以下条件，才允许进入下一阶段：

1. 团队能够用一句话准确描述本项目当前的目标视觉方向。
2. 后续生成资源时，能够明确判断某张图是否“风格跑偏”。
3. 已经形成核心资产生成顺序，后续不再无序补图。
4. Codex 在执行下一阶段时，不需要再重新猜测整体风格。

### 阶段 1 完成判断

当前阶段 1 可以视为已完成，原因如下：

1. 核心视觉方向已经明确，不再停留在“明亮一点”这类模糊描述。
2. 主体、背景、障碍、UI、特效之间的主次关系已经被明确约束。
3. 角色、敌人、浮游武器、魔法能量弹、障碍已经具备基础尺寸基准。
4. 玩家与敌人的主辅配色关系已经建立，后续不会再随机漂移。
5. 地面、障碍、描边、阴影、HUD 的关键表现规则已经确认。
6. 命中、死亡、血条、分数、Game Over 的第一轮反馈规则已经确认。
7. 后续资源生成顺序已经明确，可按阶段稳定推进。

### 进入阶段 2 的开始条件

进入阶段 2 前，执行上应遵守以下条件：

1. 不再继续扩写阶段 1 的通用风格描述，转而开始服务具体资产落地。
2. 阶段 2 只处理灰模、低细节基底和配色验证，不同时推进正式角色、敌人和 UI 重做。
3. 阶段 2 的重点是验证画面秩序，而不是生成最终插画。
4. 阶段 2 的任何出图或替换工作，都必须服从阶段 1 已确认的规则。
5. 若阶段 2 过程中发现阶段 1 某条规则明显不适用，应回写修订，而不是临时绕过。

## 阶段 2：先做灰模和配色，不急着做正式插画

### 阶段目标

阶段 2 的目标是用低细节、统一色块、简单形状验证画面秩序。这个阶段可以重做地面、障碍和边框，但它们仍应被视为“灰模与配色验证版”，不是最终正式插画。

这一阶段不追求“场景变丰富”，而是优先解决以下问题：

1. 地面过于抢眼
2. 障碍风格不统一
3. 边框存在感干扰战斗区

### 本阶段优先工作

1. 用低细节地面 tile 验证背景是否足够安静。
2. 用低细节障碍块验证阻挡物是否清楚且不抢主体。
3. 用弱化边框验证场地边界是否清楚但不干扰画面。
4. 用截图对比确认玩家、敌人、浮游武器和魔法能量弹的识别空间是否被释放。

只有当灰模和配色验证稳定后，才适合进入阶段 3，正式重做核心资产。

### 已确认的执行边界

当前已确认的阶段 2 执行边界如下：

1. 阶段 2 产物定位：可沿用的低细节资源，后续可继续打磨。
2. 阶段 2 必须截图验收，没有截图不算完成。
3. 阶段 2 允许调整 Godot 场景摆放，只要服务画面减噪和可读性。

执行约束：

1. 阶段 2 的资源不应被视为一次性废弃占位图，应尽量具备后续打磨价值。
2. 任何资源替换或场景摆放调整，都必须通过截图判断是否真正释放了主体识别空间。
3. 允许调整场景摆放，但不得借此改变核心玩法结构或绕过关卡设计问题。

### 已确认的场景基底规则

当前已确认的阶段 2 规则如下：

1. 地面 tile 基础设计尺寸：`256x256`
2. 地面块面感：弱一些，更自然柔和
3. 地面图案复杂度：很简单，只做轻微变化
4. 地面颜色偏向：偏柔和草绿
5. 障碍物类型方向：细长遗迹石柱
6. 障碍物画布设计尺寸：`20x80`
7. 障碍物底部接地 / 碰撞参考尺寸：`14x18`
8. 障碍物轮廓风格：细长、略带 3/4 顶面，可见少量前侧 / 右侧厚度
9. 障碍物表面细节密度：很低，只保留基础分块和裂纹
10. 障碍物边缘磨损程度：较明显，更像古老遗迹残柱
11. 障碍物光照与落地：左上来光，右下接地阴影，必须明确落在地面上
12. 场景边框材质感：基本不保留，尽量像弱提示线
13. 场景边框厚度存在感：中等薄，能看出边界
14. 地面与障碍的风格统一程度：强统一，看起来明显属于同一套世界

执行约束：

1. 地面应当承担“铺底”和“衬托主体”的职责，而不是单独成立为视觉中心。
2. 障碍物需要保留遗迹感，但细节密度必须受控，不能重新走向廉价拼贴感。
3. 石柱的碰撞与排序基准只取底部接地部分，不以整张贴图外轮廓作为阻挡范围。
4. 石柱必须有统一左上来光和偏右下接地阴影，不能只有高光没有落地阴影。
5. 边框只负责边界提示，不承担装饰作用。
6. 地面与障碍必须共享同一套世界观和处理语言，不能再出现素材来源割裂感。

### 已确认的场景落地规则

当前已确认的场景落地、对比控制与摆放语言如下：

1. 地面与障碍的明度差强度：差值明显，障碍一眼压住地面
2. 地面与障碍的色相差强度：中等差值，保持统一世界观下的区分
3. 战斗区域空白感倾向：更干净，明显留出战斗空间
4. 地面装饰性点缀：第一轮不允许，尽量纯净
5. 第一轮地面变体 tile：不做，先只用单一基础 tile
6. 障碍摆放语言：更自然散落，像遗迹现场
7. 障碍组合密度倾向：偏疏，给移动和射击更多空间
8. 边框颜色存在感：中等弱，能看出但不突出
9. 边框透明/弱化程度：明显弱化，尽量不打断画面
10. 第一轮小型装饰物：允许极少量，但只能是很弱的小草 / 碎石

执行约束：

1. 障碍必须比地面更明确，但不能比角色和战斗特效更抢眼。
2. 第一轮场景基底应保持极简，不通过大量地面变体和点缀来解决空旷感。
3. 障碍摆放可以自然散落，但必须保留足够移动和射击空间。
4. 小型装饰物只允许作为极弱背景信息，不能成为新的视觉噪音来源。
5. 边框要能提示边界，但不能重新形成强装饰框。

### 阶段 2 完成判断

当前阶段 2 可以视为规划层面已完成，原因如下：

1. 地面 tile 的尺寸、颜色、复杂度、块面感已经确认。
2. 障碍物的轮廓、细节、磨损、密度、摆放语言已经确认。
3. 边框的材质感、厚度、颜色存在感和弱化程度已经确认。
4. 第一轮是否使用地面变体、点缀、小型装饰物已经确认。
5. 灰模和配色验证的核心目标已经明确：安静、统一、干净，为战斗主体让位。

### 进入阶段 3 的开始条件

进入阶段 3 前，执行上应遵守以下条件：

1. 阶段 2 的第一轮产出只应包含低细节地面、障碍、边框和极少量弱装饰。
2. 不在阶段 2 中同时重做玩家、敌人、浮游武器、魔法能量弹、HUD 或战斗特效。
3. 新场景基底必须先通过截图验收，确认主体识别空间被释放出来。
4. 若地面仍然抢眼，应优先降对比和降纹理，而不是继续增加细节。
5. 若障碍不够清楚，应优先加强明度差和轮廓，而不是增加复杂花纹。

## 阶段 3：优先重做 6 个核心资产，不要全量铺开

### 阶段目标

在灰模和配色验证通过后，只重做最影响观感的 6 个核心资产，不启动全量美术替换。地面 tile 与障碍物已在阶段 2 完成，不再纳入本阶段重做范围。

### 核心资产范围

第一轮只允许推进以下 6 类资产：

1. 玩家主体与浮游魔法短杖
2. 基础敌人主体
3. 魔法能量弹
4. 命中特效
5. 敌人死亡特效
6. HUD 基础面板

### 执行约束

1. 不在本阶段增加按钮状态、复杂装饰图标、多类型敌人。
2. 每个资产都必须先满足阶段 1 的尺寸、配色、描边和可读性规则。
3. 每个资产放入游戏截图后验收，不能只看单张图是否好看。
4. 玩家第一轮不做复杂多方向动画，但玩家移动和浮游武器射击优先尝试 4 帧横向 strip 单文件；如果生成或接入效果不好，再改用每帧独立 PNG 等替代方案。
5. 基础敌人第一轮仍可制作简单 2 帧动画，不强制扩展多方向动画。

### 已确认的核心资产方向

当前已确认的阶段 3 方向如下：

1. 玩家主体身份感：传统小法师，轻装、可爱，保留明显脸部五官。
2. 基础敌人类型：小史莱姆 / 果冻怪。
3. 玩家不握持武器，手部可以简化、藏在袖口或斗篷中。
4. 玩家移动动画：4 帧轻快小跑，并带有斗篷 / 帽檐轻微摆动。
5. 玩家武器：浮游魔法短杖，围绕玩家中心以固定半径 360 度环绕，位置和朝向跟随鼠标瞄准角度。
6. 武器射击动画：4 帧，重点表现发光蓄能、轻微后坐和枪口闪光。
7. 魔法能量弹视觉类型：明亮、方向性明确，清楚表达飞行方向和命中路径。
8. HUD 基础面板实现方式：血条 / 面板用 Godot 样式，少量图标用贴图。
9. 6 个核心资产执行顺序：严格固定。

严格执行顺序：

1. 玩家主体与浮游魔法短杖
2. 基础敌人主体
3. 魔法能量弹
4. 命中特效
5. 敌人死亡特效
6. HUD 基础面板

执行约束：

1. 玩家主体贴图不需要表现握持武器，攻击来源由浮游短杖、枪口点和魔法能量弹共同表达。
2. 浮游短杖必须能独立成立为围绕玩家旋转的法器，而不是必须被手握住的枪械。
3. 魔法能量弹应优先保证飞行方向和命中路径清楚。
4. 小史莱姆 / 果冻怪需要和红橙危险配色结合，避免变成无害装饰物。
5. HUD 面板不强制全部使用贴图，优先选择更稳定、更易调的 Godot 样式实现。

## 阶段 4：先统一“形”，再统一“纹理”

### 阶段目标

先统一角色、敌人、障碍、浮游武器和魔法能量弹的轮廓、比例和识别逻辑，再考虑纹理细节。

### 执行约束

1. 轮廓不清楚的资产，不允许通过加纹理来补救。
2. 玩家、敌人、浮游武器和魔法能量弹必须在实际游戏尺寸下可辨。
3. 纹理只能服务体积和材质，不允许成为新的视觉噪音。

### 已确认的验收边界

1. 角色剪影可读性不作为硬性验收标准，彩色状态清楚即可。
2. 本阶段仍要求优先检查形状、比例和色块关系，不应提前追求复杂纹理。

## 阶段 5：场景层面做减法

### 阶段目标

在核心资产开始替换后，继续从场景层面压低背景噪音，确保战斗主体始终优先。

### 已确认规则归档

以下已确认内容也服务于本阶段：

1. 场景背景存在感：尽量低调，优先服务战斗可读性
2. 场景边框/围栏表现：弱化处理，仅作为边界提示
3. 战斗区域空白感倾向：更干净，明显留出战斗空间
4. 地面装饰性点缀：第一轮不允许，尽量纯净
5. 第一轮小型装饰物：允许极少量，但只能是很弱的小草 / 碎石

本阶段已有部分规则确认，但尚未作为独立工作阶段展开。

## 阶段 6：重新定义角色可读性

### 阶段目标

专门检查玩家、浮游武器、敌人和魔法能量弹在真实游玩尺寸、移动状态和战斗混乱情况下是否一眼可辨。

### 已确认规则归档

1. 玩家标准设计尺寸：`48x48` 左右
2. 基础敌人标准设计尺寸：`40x40` 到 `44x44` 左右
3. 魔法能量弹标准设计尺寸：`12x12` 左右
4. 玩家主色：蓝色系
5. 玩家辅助色：白色 / 浅灰
6. 基础敌人主色：红橙系
7. 基础敌人辅助色：深灰 / 黑灰
8. 浮游魔法短杖单帧建议尺寸：`32x32` 或 `40x40` 起步，默认朝向本地 `+X`

### 已确认的角色可读性边界

1. 玩家主体不做 360 度旋转，最多允许左右镜像；攻击方向由浮游魔法短杖、枪口点和魔法能量弹表达。
2. 敌人朝向不强求，敌人位置和移动方向能表达追击关系即可。
3. 彩色状态下玩家主体、浮游魔法短杖、敌人和魔法能量弹必须清楚可辨。

## 阶段 7：补统一的反馈特效

### 阶段目标

通过命中、死亡、受击等反馈，让 demo 从“能跑”提升到“像游戏”。

### 已确认规则归档

1. 命中特效主色倾向：黄白色，经典直接
2. 墙体命中特效：与敌人命中特效区分开，敌人命中更亮，墙体命中更克制
3. 敌人死亡特效表现类型：碎片散开感
4. 敌人死亡特效持续感受：很短促，干脆利落
5. 枪口火花：暂不纳入第一轮，先把命中和死亡反馈做好
6. 玩家受伤反馈：纳入第一轮，至少有闪红 / 闪白或短暂变色。
7. 敌人受击闪白 / 闪红：纳入第一轮，强化命中感。
8. 屏幕震动：暂不纳入第一轮，避免干扰画面判断。
9. 伤害数字：不纳入本轮，保持画面干净。

### 当前状态

本阶段的第一轮反馈边界已经确认，后续执行时应优先完成命中、死亡、玩家受伤和敌人受击闪烁，不加入屏幕震动和伤害数字。

## 阶段 8：最后再处理 UI，而不是最早

### 阶段目标

在玩法画面稳定后，再统一 HUD、血条、分数和 Game Over。UI 不应先行定义整套视觉，而应从属于玩法画面。

### 已确认规则归档

1. UI 风格：简洁、低装饰、以清晰易读为主
2. HUD 布局气质：极简，以数字和血条清晰为第一优先级
3. HUD 字体气质：圆润可爱，轻松一点
4. HUD 数字信息位置倾向：右上角，传统稳妥
5. 血条表现方式倾向：底部居中横向血条，最直接
6. 血条显示方式：不显示数字，只保留图形血条
7. 分数字号存在感：中等，清楚但不抢画面
8. Game Over 面板视觉强度：偏弱，尽量保持整体克制
9. HUD 基础面板实现方式：血条 / 面板用 Godot 样式，少量图标用贴图
10. 主菜单纳入本轮 UI 处理，并一起统一视觉风格

### 当前状态

本阶段已有主要规则确认，但应在阶段 3 到阶段 7 稳定后再正式执行。主菜单需要纳入同一轮 UI 统一，但不应早于局内 HUD 和 Game Over 的基础统一。

## 阶段 9：建立 Codex 执行方式，不要直接“帮我做一套美术”

### 阶段目标

把后续美术提升拆成可执行、可验收、可回滚的小任务，避免一次性批量生成导致整体失控。

### 推荐执行方式

1. 后续每个执行聊天只处理一个阶段，避免混乱。
2. 每次执行前先确认该阶段任务清单。
3. 每次执行后必须截图验收。
4. 每个阶段执行后新增阶段总结文档，规划文档保持稳定。
5. 单张资源不能脱离游戏截图判断质量。
6. 如果生成结果不好，先截图复盘问题，再决定迭代还是回退。

### 当前状态

本阶段的协作原则已经确认，后续仍需要在实际执行聊天中为每个阶段生成对应任务清单和验收清单。
