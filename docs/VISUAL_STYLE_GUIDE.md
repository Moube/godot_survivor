# 项目视觉风格基准

## 文档目的

本文档用于定义本项目当前阶段的统一视觉规则，供后续 Codex 在生成、替换和调整美术资源时直接遵循。
本文件服务于“先锁视觉规则，再逐阶段执行”的开发方式，不负责描述具体执行任务顺序。

---

## 风格定义

本项目采用 **明亮可爱卡通风**，整体是 **轻微斜俯视、带一点 3/4 感的奇幻 Top-Down Shooter**。

目标观感：

- 可爱、圆润、卡通化
- 明亮但不过曝
- 低纹理、大色块、干净
- 明显粗描边
- 战斗主体一眼可辨
- 场景服务玩法，不与玩法抢注意力

不追求的方向：

- 写实材质
- 高纹理密度
- 重装饰 fantasy UI
- 多来源拼贴感
- 背景比角色更抢眼

---

## 总体视觉基调

### 视角与空间

- 视角表现：轻微斜俯视，带一点 3/4 感
- 角色体量占屏比例：中等比例，兼顾场景空间感
- 场景边框/围栏：弱化处理，仅作为边界提示

### 光照与明暗

- 光照表现：整体明亮，但保留一定体积感和层次
- 来光方向：左上来光，阴影偏右下
- 阴影强度：中等，能看出层次但不压暗整体
- 不允许使用写实重渐变和复杂材质高光

### 轮廓与纹理

- 描边颜色：接近黑灰色
- 描边粗细：偏粗，强化卡通感
- 纹理密度：低纹理、大色块、整体干净
- 优先保证轮廓和色块识别，不用纹理堆信息

### 颜色气质

- 整体饱和度：中等饱和，更柔和耐看
- 角色、敌人、子弹、特效优先于背景获得颜色注意力
- 地面、边框、障碍的饱和度和对比必须低于战斗主体

---

## 玩法可读性原则

后续所有资源必须服从以下可读性优先级：

1. 玩家
2. 敌人
3. 子弹与战斗特效
4. 障碍物
5. 地面与边框
6. UI 装饰

执行要求：

- 玩家、敌人、子弹、命中特效必须比背景更容易被第一眼识别
- 战斗过程中，背景只能提供环境基底，不能形成视觉竞争
- 单张资源是否“好看”不重要，放进游戏截图后是否清楚才重要

---

## 角色与敌人基准

### 玩家

- 角色气质：可爱中带一点小帅气、小冒险感
- 主体身份感：小冒险者，轻装、灵活
- 主色：蓝色系
- 辅助色：白色 / 浅灰
- 标准设计尺寸：`48x48` 左右
- 需要明显武器，但武器不能遮挡角色主体
- 玩家朝向不强求必须明显，只要武器 / 子弹方向能表达攻击方向即可

### 基础敌人

- 第一轮基础敌人类型：小史莱姆 / 果冻怪
- 主色：红橙系
- 辅助色：深灰 / 黑灰
- 标准设计尺寸：`40x40` 到 `44x44` 左右
- 危险感优先于配色丰富度
- 敌人朝向不强求必须明显，只要位置和移动方向能表达追击关系即可

### 玩家与敌人的关系

- 必须形成明显冷热对比
- 必须一眼分辨阵营，不允许配色漂移到同类区间
- 彩色状态下，玩家、敌人、子弹三者必须清楚可辨
- 剪影可读性可作为参考，但不作为硬性验收标准

---

## 子弹与战斗物件基准

- 子弹存在感：明显，保证飞行物和战斗节奏清楚
- 子弹视觉类型：物理弹丸，明确射击感
- 标准设计尺寸：`12x12` 左右
- 子弹要优先服务方向识别与命中路径识别
- 子弹不能小到被背景吞掉，也不能花到喧宾夺主

---

## 场景基底基准

### 地面

- 材质方向：柔和的泥土 / 草地混合地面
- 颜色偏向：偏柔和草绿
- 明度倾向：中等明度
- 饱和度倾向：中低饱和，保留一点生气
- tile 基础设计尺寸：`256x256`
- 块面感：弱一些，更自然柔和
- 图案复杂度：很简单，只做轻微变化
- 第一轮不做地面变体 tile
- 第一轮不允许地面装饰性点缀

### 障碍物

- 材质方向：石块 / 遗迹方砖类奇幻障碍
- 标准设计尺寸：`64x64` 左右
- 轮廓风格：中度规整，带一点自然破损
- 表面细节密度：很低，只保留基础分块和裂纹
- 边缘磨损程度：较明显，更像古老遗迹残块
- 明度倾向：偏暗一些，强调厚重和阻挡感
- 饱和度倾向：低饱和，明显退后

### 地面与障碍的关系

- 风格统一程度：强统一，看起来明显属于同一套世界
- 明度差强度：差值明显，障碍一眼压住地面
- 色相差强度：中等差值，保持统一世界观下的区分
- 障碍必须比地面更明确，但不能比角色和战斗特效更抢眼

### 边框与战斗区域

- 边框材质感：基本不保留，尽量像弱提示线
- 边框厚度存在感：中等薄，能看出边界
- 边框颜色存在感：中等弱，能看出但不突出
- 边框透明 / 弱化程度：明显弱化，尽量不打断画面
- 战斗区域空白感：更干净，明显留出战斗空间
- 障碍摆放语言：更自然散落，像遗迹现场
- 障碍组合密度：偏疏，给移动和射击更多空间
- 第一轮只允许极少量很弱的小草 / 碎石作为装饰

---

## 特效反馈基准

- 特效整体方向：明亮夸张，强化命中与死亡反馈

### 命中特效

- 主色倾向：黄白色，经典直接
- 墙体命中特效需要与敌人命中特效区分
- 敌人命中更亮，墙体命中更克制

### 敌人死亡特效

- 表现类型：碎片散开感
- 持续感受：很短促，干脆利落

### 第一轮反馈边界

- 纳入：玩家受伤反馈
- 纳入：敌人受击闪白 / 闪红
- 暂不纳入：枪口火花
- 暂不纳入：屏幕震动
- 不纳入本轮：伤害数字

---

## UI 基准

### 总体方向

- UI 风格：简洁、低装饰、以清晰易读为主
- 不走木牌、卷轴、宝石堆砌路线
- UI 从属于玩法画面，不应先行决定整套视觉存在感

### HUD

- HUD 布局气质：极简
- 字体气质：圆润可爱，轻松一点
- 数字信息位置：右上角
- 血条位置与形式：底部居中横向血条
- 血条不显示数字，只保留图形血条
- 分数字号存在感：中等，清楚但不抢画面

### 面板与实现方式

- Game Over 面板视觉强度：偏弱，尽量保持整体克制
- HUD 基础面板实现方式：血条 / 面板优先使用 Godot 样式，少量图标可用贴图
- 主菜单纳入本轮 UI 统一范围，但不应早于局内 HUD 基础统一

---

## 第一轮资源制作约束

- 第一轮允许玩家和基础敌人制作简单 2 帧动画
- 不做复杂多方向动画
- 不做多类型敌人扩展
- 不做高复杂度装饰图标
- 不做高密度纹理版本

---

## 一票否决项

出现以下情况时，应判定资源方向跑偏：

- 背景比玩家、敌人、子弹更抢眼
- 地面出现高对比、大面积亮斑或复杂重复纹理
- 障碍物过白、过亮或细节密度过高
- 玩家与敌人的颜色区分不明显
- 描边风格在不同资源间不统一
- UI 装饰感明显强于场景主体
- 单张资源精致，但放进游戏画面后主体识别下降
