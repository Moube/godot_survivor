# 阶段3：敌人与追踪完成记录

## 阶段目标

本阶段的目标是让敌人进入场景，并形成最小可验收的追击闭环：

1. 创建可复用的 `Enemy.tscn`
2. 敌人进入关卡后持续朝玩家移动
3. 不再依赖手动摆放敌人，而是由关卡自动刷怪
4. 多个敌人同时存在时逻辑保持稳定

## 本阶段完成内容

### 1. 敌人场景完成基础搭建

`scene/enemy/Enemy.tscn` 已完成最小可运行结构：

- 根节点：`CharacterBody2D`
- 子节点：`CollisionShape2D`
- 子节点：`Polygon2D` 作为敌人主体占位图形
- 子节点：`Polygon2D` 作为朝向标记

当前仍然使用临时图形进行可视化，便于先完成功能验证。后续如果接入正式素材，可以直接替换显示节点。

### 2. 敌人追踪逻辑完成

`scene/enemy/Enemy.cs` 已实现基础追击行为：

- 导出 `MoveSpeed` 作为移动速度
- 导出 `TargetGroupName` 用于按分组查找目标
- 导出 `StopDistance` 作为贴近玩家后的停靠距离
- 在 `_PhysicsProcess()` 中持续寻找并缓存玩家目标
- 根据敌人与玩家的方向计算移动速度
- 根据追击方向同步更新敌人朝向

当前实现默认查找 `player` 分组，而玩家场景在前序阶段已经加入该分组，因此敌人进入场景后可以直接开始追击。

### 3. 关卡改为自动刷怪

`scene/level/Level01.cs` 不再依赖手动放置敌人，而是改为由关卡负责自动生成：

- 新增导出的 `EnemyScene`
- 新增 `EnemySpawnIntervalSeconds` 控制刷怪间隔
- 新增 `MaxEnemyCount` 控制场上敌人数上限
- 新增 `MinSpawnDistanceFromPlayer` 控制与玩家的最小出生距离
- 新增 `SpawnAreaHalfExtents` 控制刷怪范围

关卡启动后会先生成玩家，再启动刷怪计时器，并立即刷出第一只敌人。

### 4. 刷怪位置逻辑完成

为了避免手动摆放怪物出生点，本阶段实现了简单的边缘刷怪策略：

- 在关卡矩形范围边缘随机生成候选位置
- 尝试避开距离玩家过近的位置
- 如果多次尝试仍未满足最小距离要求，则回退到最后一次候选位置

当前策略足以支撑阶段3的功能验收，同时保留了后续继续升级为更复杂 `Spawner` 系统的空间。

### 5. 关卡场景补齐刷怪节点

`scene/level/Level01.tscn` 已补充与自动刷怪逻辑对应的节点：

- `Enemies`：作为敌人实例容器
- `SpawnTimer`：作为周期刷怪计时器

同时将 `EnemyScene` 资源引用接入 `Level01.tscn`，保证运行关卡时无需手动额外配置即可刷出敌人。

## 验收结果

本阶段已完成并通过以下验收项：

1. 运行关卡后会自动生成敌人
2. 敌人进入场景后会持续朝玩家移动
3. 场景中可稳定同时存在多个敌人
4. 刷怪逻辑不依赖手动摆放敌人
